Læringsmål

Webudvikling (JS/ReactJS)

Kortfristede læringsmål (hver anden uge)

  • Uge 1-2: Forstå DOM-manipulation ved at bygge en enkel interaktiv liste, der kan tilføje og fjerne elementer.
  • Uge 3-4: Implementere fetch API for at hente data fra en offentlig API og vise det i en React-komponent.
  • Uge 5-6: Opbygge en simpel React-applikation med tilstandshåndtering ved hjælp af `useState` og `useEffect`.
  • Uge 7-8: Indføre routing med React Router og tilføje flere sider til applikationen.
  • Uge 9-10: Skabe dynamiske komponenter med prop-håndtering og løse performanceproblemer i React.

IT-Sikkerhed

Kortfristede læringsmål (hver anden uge)

  • Uge 1-2: Introduktion til kryptering ved at lære om hashing og implementere en simpel hash-funktion i kode.
  • Uge 3-4: Forståelse af XSS-angreb og implementere inputvalidering og escape-teknikker i et projekt.
  • Uge 5-6: Konfiguration af sikker kommunikation mellem klient og server ved hjælp af HTTPS og krypterede tokens.
  • Uge 7-8: Gennemføre en sårbarhedsanalyse på en testapplikation og skrive en rapport om fundne problemer.
  • Uge 9-10: Opsætning af sikkerhedsovervågning og analyse af logfiler for at identificere mistænkelig aktivitet.

Sammenkobling med langsigtede mål

De kortsigtede mål er designet til at bygge de nødvendige færdigheder og viden over tid. For eksempel:

  • Webudviklingsmål som DOM-manipulation og fetch API sikrer en stærk grundlæggelse i interaktive brugergrænseflader, hvilket understøtter ReactJS-kompetencer.
  • IT-sikkerhedsmål som XSS-beskyttelse og kryptering opbygger en forståelse for applikationssikkerhed, der er nødvendig for pentesting og systembeskyttelse.
Læringsmål

Webudvikling (JS/ReactJS)

Viden

  • Udviklingsbaseret viden om teori og metoder inden for JavaScript-programmering og front-end udvikling med ReactJS.
  • Forståelse for grundlæggende forhold i relation til JavaScript, herunder dets anvendelse i at skabe interaktive og responsive webapplikationer med dynamiske brugergrænseflader.
  • Kendskab til teknologiske begreber som DOM-manipulation, hvor man kan ændre HTML- og CSS-indhold dynamisk, samt asynkron programmering for at håndtere datatransmissioner uden at blokere brugerinteraktionen.

Færdigheder

  • Anvende fetch og async/await til at hente data fra API'er, hvilket muliggør effektiv kommunikation med servere og opdatering af brugergrænsefladen uden genindlæsning af siden.
  • Designe og implementere interaktive brugergrænseflader, der engagerer brugerne og forbedrer interaktionen med applikationen ved hjælp af React-komponenter og tilstandshåndtering.

Kompetencer

  • Håndtere udviklingsforløb for ReactJS-applikationer med fokus på struktureret kode og effektiv datahåndtering.
  • Deltage i tværfagligt samarbejde om webudvikling for at skabe brugervenlige og skalerbare løsninger, der møder kundens behov og forbedrer brugeroplevelsen.
  • Løse praksisnære problemer med fokus på at forbedre performance og kodekvalitet, hvilket sikrer hurtige indlæsningstider og vedligeholdelse af ren og effektiv kode.

IT-Sikkerhed

Viden

  • Udviklingsbaseret viden om de anvendte praksisser, teori og metoder inden for IT-sikkerhed.
  • Forståelse for sikkerhedsvurdering, risikovurdering, sårbarhedsanalyse, og deres anvendelse.
  • Forståelse for teknologiske begreber som netværkssikkerhed, applikationssikkerhed, kryptering og social engineering.

Færdigheder

  • Anvende centrale metoder og redskaber inden for pentesting til at identificere og rapportere sårbarheder.
  • Anvende moderne pentesting-teknikker og værktøjer som Burp Suite.
  • Udvikle forsvarsmekanismer mod sårbarheder, herunder og sikkerhedsovervågning.
  • Systematisk sårbarhedsanalyse og rådgivning om sikkerhedsforbedringer.
  • Vurdere praksisnære problemstillinger og opstille løsninger inden for IT-sikkerhed og pentesting.

Kompetencer

  • Deltage i samarbejde om IT-sikkerhed og pentesting i tværfaglige projekter.
  • I en struktureret sammenhæng tilegne sig ny viden og færdigheder inden for IT-sikkerhed og pentesting.
